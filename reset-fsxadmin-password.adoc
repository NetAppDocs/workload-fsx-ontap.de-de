= Setzen Sie das fsxadmin-Passwort in NetApp Workload Factory zurück
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Setzen Sie das fsxadmin-Passwort in NetApp Workload Factory bei Bedarf zurück.

Wenn Sie während der Erstellung des Dateisystems einen alternativen Benutzer angegeben haben, können Sie nur das fsxadmin-Passwort in der AWS-Konsole zurücksetzen.

.Schritte
. Melden Sie sich mit einem der link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["Konsolenerfahrungen"^]an.
. Wählen Sie das Menü aus image:workloads-menu-icon.png["Über das Hamburger-Menüsymbol kann man zu Workloads wie Speicher, EDA, KI, Datenbanken, VMware und Administration navigieren."] und wählen Sie dann *Speicher* aus.
. Wählen Sie im Speichermenü *FSx für ONTAP* aus.
. Wählen Sie unter *FSx for ONTAP* das Aktionsmenü des Dateisystems aus, für das Sie das fsxadmin-Passwort zurücksetzen möchten, und wählen Sie dann *Verwalten*.
. Wählen Sie in der Dateisystemübersicht das Aktionsmenü aus.
+
image:screenshot-reset-fsxadmin-password.png["Ein Screenshot des Aktionsmenüs auf der Registerkarte „Übersicht“ für ein Dateisystem."]

. Wählen Sie *fsxadmin Passwort zurücksetzen*.
. Geben Sie im Dialogfeld Passwort für fsxadmin zurücksetzen ein neues Passwort für fsxadmin ein und bestätigen Sie es erneut.
. Wählen Sie *Anwenden*.
